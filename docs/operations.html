Betrieb und Admin-Abläufe

Zurück zur Übersicht

Lokaler Start

python -m venv .venv
. .venv/bin/activate
pip install -r requirements-dev.txt
npm install
python src/manage.py migrate
python src/manage.py runserver

Beim ersten Aufruf führt die App durch die Ersteinrichtung und legt den ersten Admin an.

Docker

cp .env.example .env
docker compose up --build

Danach läuft die App unter http://localhost:8000.

Typischer Admin-Ablauf

  1. Lager anlegen und Lager-Fördersatz pflegen.
  2. Preisverwaltung öffnen und Lagerpauschalen für Teilnehmer/Begleiter und 1/2 Wochen eintragen.
  3. Getränke- und Essenspreise als Preisregeln anlegen.
  4. Teilnehmer importieren oder manuell anlegen.
  5. PINs setzen oder Teilnehmer im Kiosk ihre PIN beim ersten Login setzen lassen.
  6. Während des Lagers Kiosk für Buchungen nutzen.
  7. Abrechnung prüfen und Exporte erzeugen.

Tests

.venv/bin/python src/manage.py check
.venv/bin/python -m pytest
npm run test:e2e
npm run test:local

Die Tests decken Authentifizierung, Rollen, Import, Abrechnung, Kiosk und Preisverwaltung ab.
