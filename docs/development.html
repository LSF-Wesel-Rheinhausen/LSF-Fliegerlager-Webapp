pre-commit bündelt Basischecks, Ruff und gitleaks. Die Hooks werden mit pre-commit install aktiviert.

Git-Workflow pro Arbeitspaket

  1. GitHub-Zugang mit gh auth status prüfen und bei Bedarf mit gh auth login -h github.com reparieren.
  2. Immer zuerst einen frischen Feature-Branch von master anlegen.
  3. Push 1 vor Start der eigentlichen Umsetzung ausführen: Branch sofort mit git push -u origin <feature-branch> remote absichern.
  4. Implementierung, Tests, Coverage-Arbeit und Review-Fixes ausschließlich auf diesem Branch durchführen.
  5. Push 2 nach Abschluss ausführen: final committen, relevante Checks erfolgreich abschließen und denselben Branch erneut pushen.

Direktes Arbeiten oder Pushen auf master ist nicht der Standardprozess.

Django-ORM und Transaktionen

  • Iterierte ForeignKey- und OneToOne-Daten mit select_related() laden.
  • Reverse ForeignKeys und ManyToMany-Daten mit prefetch_related() vorbereiten.
  • Finanzielle Schreibvorgänge und Kiosk-Buchungen mit transaction.atomic() schützen.
  • Raw SQL vermeiden und die ORM-Schnittstellen bevorzugen.

Tests

  • Tests dürfen keine externen Netzwerke, echten APIs oder echten Mailversand nutzen.
  • Externe Abhängigkeiten werden mit unittest.mock oder pytest-mock gepatcht.
  • Neue wiederverwendbare Testdaten werden über factory_boy in tests/factories.py erzeugt.
  • Import-, Export-, Permission-, Kiosk- und Abrechnungspfade brauchen Edge-Case-Tests.

Security und Datenschutz

  • Keine PII, Zahlungsdetails, PINs, Passwörter oder Secrets loggen.
  • Keine .env-Dateien committen; nur .env.example mit Platzhaltern pflegen.
  • Für sicherheitsrelevante Zufallswerte das Modul secrets verwenden.
  • Dateiuploads nicht anhand der Endung allein verarbeiten; Inhalt und Format validieren.

Frontend-Konventionen

  • Servergerenderte Templates nutzen semantisches HTML und beschriftete Formularfelder.
  • Layouts sind mobile-first und basieren auf CSS Grid/Flexbox.
  • Inline-Styles und jQuery sind nicht vorgesehen.
  • Dynamisches Verhalten wird mit modernem Vanilla JavaScript oder bewusst ausgewählten leichten Tools umgesetzt.
