This bugfix resolves an issue in the deployment agent where an automatic update would fail if the `.env` file was not present on the host system. Docker Compose creates missing mounted files as directories by default, which caused `docker compose up` in the deployment script to crash with `stderr: /deployment/.env is a directory`.
The deployment agent now auto-detects the `env_file` property configured in the `docker-compose.yml`, so it seamlessly supports custom env files like `stack.env` used in Portainer or other environments, and correctly checks if the configured environment file exists before appending the `--env-file` argument.
