This bugfix resolves an issue in the deployment agent where an automatic update would fail if the `.env` file was not present on the host system. Docker Compose creates missing mounted files as directories by default, which caused `docker compose up` in the deployment script to crash with `stderr: /deployment/.env is a directory`. The deployment agent now correctly checks if the configured `ENV_FILE` is an actual file before appending the `--env-file` argument.
