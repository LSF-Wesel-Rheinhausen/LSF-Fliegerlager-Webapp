# PR Changelog

Dieser Ordner sammelt kurze Zusammenfassungen pro Pull Request.

Konvention:

- Eine Datei pro PR oder Branch.
- Dateiname: `pr-<nummer>-<kurzer-titel>.md`, sobald eine PR-Nummer existiert.
- Bis zur PR-Erstellung: `<branch-name>.md` mit `/` durch `-` ersetzt.
- Jeder Eintrag enthaelt Zusammenfassung, geaenderte Dateien, Tests und offene Punkte.
- Der CI-Check verlangt einen Eintrag, sobald eine Datei unter `src/` geaendert wird; das gilt auch fuer Modul-Dokumentation in diesem Verzeichnis.
